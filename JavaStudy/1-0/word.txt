◆IT
・IT
 「Information Technology」の略称。「情報技術」と訳される。コンピューターやネットワークといった情報処置に関する技術の総称。
・ICT
 「Information and Communication Technolgy」、「情報伝達技術」、「情報通信技術」を指す。「ヒトとヒト」、「ヒトとモノ」の間で情報や知識を共有することを指す。
 スカイプやLIVE配信システムがここにあたる。
・IoT
 「Internet of Things」の略称。「モノのインターネット」と呼ばれる。「身の回りのあらゆるモノがインターネットにつながる」仕組みのこと。
 モノが相互に通信し、遠隔からも認識や計測、制御など可能になる。

◆OS
・OS
 「Operating System」(オペレーティングシステム)の略称。直訳は「動作システム」。スマホなどパソコンを通常通りに動作させるシステムのこと。
 スマホの画面タッチを認識したり、PCのキーボードを認識したり、それを反映させたりを全てOSが実現している。
・Windows
 Microsoft社が販売しているOSのこと。Linuxをベースに作成。
・macOS(Mac OS X)
 Apple社のOSを指す。Unixをベースに作成。
・UNIX
 OSの一種でもっとも古いOSの一つ。安全性が高く、動作が軽いこと、安価なことが特徴。
・Linux
 カスタマイズすることのできるOSのこと。LinuxはOSの中核に存在。
 リーナス・トーバルズが発明。Linus UnixのようなOS→Linux

◆言語・環境
・マークアップ / フロントエンド
 マークアップエンジニアは適切な構造化、SEO(Webサイトの成果を向上させる施策のこと)などコーディングスキルが高いエンジニア。
 フロントエンドエンジニアは、Webデザイナーが作ったデザインを再現し、ブラウザに表示させるエンジニア。 年々、仕事領域が広まってきている。
　・HTML
 「Hyper Text Markup Language」(ハイパーテキストマークアップランゲージ)の略称。文章の見出しをつけたり、表を作ったり。文章をタグで囲い込むことでコンピュータが認識する。
　・CSS
 「Cascading Style Sheets」(カスケーティングスタイルシート)の略称。
  構造に対して、どのような装飾をするか。色や配置など。
　・JavaScript
 Webサイトを表示する上で動くプログラム。写真のスライダーや、画像の拡大などで使用される。
・サーバーサイド
 サーバー側で動作するプログラミングの開発やデータ処理など。
　・.NET(C#, VB)
 Microsoftが開発したプログラミング言語。.NET Framework上で稼働するWindowsアプリの開発現場でよく利用される。
　・Java
 OSやハードウェアに依存せずに動作する。さまざまな大規模システムで利用される。
　　・Java SE
 Java動作の汎用的な機能をまとめたもの。
　　・Java EE(Jakarta EE)
  Javaの開発および実行環境、標準仕様のこと。Java EEを利用するにも基本機能としてJava SEが必要。
　・Perl
 サーバーサイドでの処理を実装できるプログラミング言語。
　・PHP
 Hypertext Preprocessorの略称。動的なコンテンツの作成に使われる。閲覧者の属性や時間帯によって表示される内容が変わってくるコンテンツ。
　・Ruby
 1995年にまつもとひろゆきが作成した日本産のプログラミング言語。主に、WebアプリケーションやWebサービスの開発に利用される。コンパイルがいらないスクリプト言語。

◆SQL
・SQL
「Structured Query Language」の略称。直訳で構造化問い合わせ言語。データベースを操作するときに使う。データの保存ができ、何億、何兆とデータを保存しておける。
　・DB2
 リレーショナルデータベース管理システム。
　・MySQL
 オープンソースのリレーショナルデータベース管理システム。
　　・Maria DB
 MySQLから派生したRDBMS（リレーショナルデータベース管理システム）
　・Oracle DB
 上記同様データ管理システム
　・PostgreSQL
 リレーショナルデータベース管理システムのひとつ。オープンソースのため利用料はかからない。
　・SQLite
 ファイルベースのデータベースエンジン。複数のクライアントから操作する場合などはMySQLやPostgreSQLを利用することが多い。
　・SQL Server
 Windows OSで動作させることができるリレーショナルデータベースの一種です。
・CRUD
 Create（生成）、Read（読み取り）、Update（更新）、Delete（削除）の頭文字をとった言葉。
・トランザクション
 企業の情報システムなどが扱うデータの種類の一つで、業務に伴って発生した出来事の詳細を記録したデータ。データベースにおける一連の処理。
　・コミット
 トランザクション（データベースにおける一連の更新処理）が全て正常に完結したという宣言。
　・ロールバック
 データベースにおけるトランザクション処理の過程で障害が発生した場合、障害発生の前の処理過程に戻ること。
・排他制御(排他ロック)
 他のユーザーからファイルやデータベースを同時に利用されデータの整合性がおかしくならないように、データの読み書きやアクセスを一時的に制御すること
　・楽観ロック
 同時アクセスによるデータの不整合を防ぐ排他制御（ロック）の手段
　・悲観ロック
 更新対象データを取得時にロックして他者が操作できないようにする方式。ECサイトを例にすると商品が残り1つだった場合Aさんが閲覧中の場合、Bさんはその情報を取得できない。
・実行計画
 実行計画とはユーザが問い合わせたクエリ（SQL文）を実行する手順書。「どの順番にテーブルを結合するか」「ソート方法」「検索方法』などの作業内容や、想定される実行コストが記述されている。
・チューニング
 
・パフォーマンス
コンピューターの処理性能や実行速度、通信回線の速度・容量のこと
・スロークエリ
 実行が遅いQuery(命令)
・クエリログ
 サーバーが実行した全てのSQLクエリを記録するログ
・IDE(統合開発環境)
 テキストエディタ・コンパイラ・デバッガなどの機能を、一つにまとめたもの。開発効率化が期待できる。
　・Eclipse
 統合開発環境とも呼ばれ、Javaの開発環境で使われる。
　・NetBeans
 統合開発環境とも呼ばれ、Javaの開発環境で使われる。Javaだけでなく、C系の言語で開発される際も利用される。
　・SpringToolSuite
 上記同様統合開発環境の一つ。Eclipseをベースにしている。
・DBツール
 データーベース管理システム上のデータを扱うためのツール。
　・A5:SQL Mk-2
 SQL文の入力支援やER図作成などの機能を備えた高機能なSQL開発環境。また、SQLの実行計画を取得したり、実行結果を「Microsoft Excel」へ出力することも可能。s
　・pgAdmin
 PostgreSQL に関する各種操作を，簡単に行えるツール
　・DBeaver
 ER図を自動生成出来たり、データのリレーションが可能。また、多種多様なDBに対応している。
　・MySQL Workbench
 MySQLの公式サイトにてMySQL Serverと共に配布されている、データベースの設計・開発・管理を一本で実現している無料ツール
　・Sequel Pro
 データベースをExcelのような表で確認・編集をすることができるMac専用のアプリ
　・TeamSQL
 データーベースの種類に関係なく横断的に使えるSQLクライアント

◆各種プロトコル
・HTTP
  「Hyper text Transfer Protocol」通信内容が暗号化されていないプロトコル 通信規約。
・FTP
 「File Transfer Protcol」の略称。ファイルをネットワーク上のある場所から別の場所に移動するための手順。
・HTTPS
 通信内容が暗号化されているプロトコル。sはSecure
・SSL
 「Secure Sockets Layer」の略称。インターネット上でデータを暗号化して送受信する仕組みのこと。Webサイト閲覧者に安心してもらう。
 SSL化していないと、データが盗まれてしまったりする。
・SSH
 「Secure Shell」の略称。ネットワークに接続された機器を遠隔操作し、管理するための手段。安全にサーバーへ接続する。
・ポート、ポート番号
  インターネットの玄関、扉。IPアドレスを建物の住所に例えた場合、何号室なのか、どのアプリケーション同士で通信するにか
・HTTP, FTP, SSHの各ポート番号
 HTTP: 80 / FTP: 20 / SSH / 22
・ファイアウォール
 外部からの不正通信を防ぐ防火壁。
・TCP
「Transmission COntrol Protocol」の略称。プロトコルの一つ。コネクション型通信ともいい、通信開始前の相手との間で仮想的な通信路を確保する。

◆IPアドレス
・ファイル
  コンピューターにおけるデータの管理単位の一つ。OSから見て最小の記録単位
　・グローバルIPアドレス
 インターネットに接続されているコンピュータや通信機器を個々に特定するための一意で割り当てられたIPアドレスのこと。255.255.255.255など
　・プライベートIPアドレス
 LANなどの企業内ネットワークに存在する端末が、自由に使用できるIPアドレスのこと。グローバルより使用範囲は狭い?

◆ドメイン
・ドメイン
 インターネットの住所。Webサイトがどこにあるのかを判別する情報。プロトコルの後に続くtest.comやhogehoge.co.jpなどの部分
・DNS
 「Domain Name System」ドメイン名とIPアドレスを対応づける仕組みのこと。hogehoge.comは127.001.001など
・VPN
 「Vitual Private Nerwork」。離れた場所の間を仮想的な専用線でつないで安全なデータ通信を実現する仕組み。社内であればLANで繋ぐ。離れたLAN同士を繋ぐ。
・ブリッジ
 IPアドレスとMacアドレスの架け橋のようなもの。Macアドレスとは機器のネットワーク機器に割り当てられる住所。特定の場所に届けるまでにはいくつものMacアドレスを通過するが、その時に使ってるのがブリッジ
・プロキシサーバー
 インターネットに直接接続できないコンピューターに代わり、インターネットに接続し、Webサイトへのアクセスなどを行うサーバーのことを指す。

◆電子メール
・SMTP
 「Simple Mail Protocol」。送信まで手順のこと。
・POP
「Post Office Protocol」。受診までの手順ここと。
・IMAP
「Internet Message Access Protocol」。メールを読み込むこむための手順。POPに近い。メール本体はサーバーに残しておいてキャッシュをダウンロードする。

◆セッションとクッキー
・セッション
 サーバー側に保存された情報。Webブラウザを閉じるまで有効。サーバーに保存されているためクッキーに比べてデータ改ざんの可能性は低い。
・クッキー
 クライアントに保存された情報。クライアントとは自身のスマホ、パソコンなど
・キャッシュ
 一度利用したデータにまたアクセスした際、高速にやりとりができるよう情報を保存しておくこと。

 複数のカラムがある中から、必要なカラムを抽出して保存。これで、高速かつわかりやすい
