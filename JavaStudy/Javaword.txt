■ 開発
ウォーターフォール開発 要件定義→システム設計→プログラミング→テストの順にその名の通り各工程を完了させてから進む開発。デメリットは手戻り(全行程に戻って作業すること)
アジャイル開発 短いスパンで反復開発を繰り返すので開発スピードが高い。設計→プログラミング→テスト このスパンが週単位で繰り返される
マイグレーション 移住や移動。既存システムやソフトウェアなどを新しいプラットフォームやシステム(高台、動作環境)に移行すること
デグレーション 悪化。修正を行ったことによって品質退化してしまうこと
デプロイ 配置、展開。サーバーを借りて展開、配置すること
コンフリクト 競合、衝突。対象データを複数が同時に更新してしまうこと。

環境
    開発環境（ローカル）自身のPC内での環境
    検証環境（ステージング）リリース予定の昨日をテストするための環境。
    本番環境 サーバーを借りて作ったコードを後悔する環境

設計書／仕様書
    要件定義 利用者にヒアリングをして、機能や性能を洗い出す。
    基本設計書 ソフトウェアの動きを外から見たさいにどういう動きをするのか(What)を決めたもの[利用者目線]
    詳細設計書 基本設計で決められた動きをどうやって実現するか(How)を決めたもの[開発者目線]

テスト
    単体テスト（UT）モジュールレベルの動作確認。プログラムの設計通りにできているか。モジュール(ひとまとまりの機能のこと) Unit Test
    結合テスト（CT／IT）モジュールを結合した状態で入出力のテスト。 Combined Test
    総合テスト（ST）システム全体の動作確認。負荷試験、耐久試験など System Test

エビデンス／証跡 テスト工程でシステムの動作などを示した書類やデータ
データパッチ patch 後付け。更新、修正などを行う。「パッチを当てる、パッチを適用する」など
Access 利用可能な状態にすること

マクロ 一連の手順を記録しておくことで操作の自動化する, 時間効率good!
    VBA 【Visual Basic for Applications】Microsoft Officeに含まれている拡張機能。複雑な処理の自動化ができるプログラミング言語

GAS（Google Apps Script）Googleが提供しているJavaScriptベースの言語。実行環境はサーバー上にある

API【Application Programming Interface】ソフトウェアの機能を外部から呼び出してアプリケーション開発を容易にする。JSON,Google,Youtube
    WEB API Web技術を用いたAPI
    REST API RESTと呼ばれる設計原則にしたがって策定されたもの
    SOAP【Simple Object Access Protocol】他のサービスやデータを呼び出すさいの通信規約プロトコル。

Ajax【Asynchronous JavaScript XML】JavaScriptとXMLを使って非同期にサーバーとの間の通信を行うこと
    XML【Extensible Markup Language】文章の見た目や構造を記述するためのマークアップ言語
    JSON【JavaScript Object Notation】軽量なデータフォーマット。
    ステータスコード HTTPリクエストの結果を表す3桁の数字
    クロスオリジン / CORS（Cross-Origin Resource Sharing）あるオリジン(プロトコル+ポート番号)で動いている Web アプリケーションに対して、別のオリジンのサーバーへのアクセスをオリジン間 HTTP リクエストによって許可できる仕組み

バリデーション 入力内容や記述内容が要件を満たしているか妥当性を確認すること

データ連携 異なるデータやシステムを繋ぐ仕組み
    Daemon デーモン。メモリで待機している常駐プログラムのUNIX系OS呼び名 メーラーデーモン
    CRON UNIX系OSで利用されるデーモンの一種。常駐プログラム
    バッチ 同類処理をまとめておく、同時処理で効率化
    ログ コンピューターの利用状況やデータ通信など履歴や情報の記録を取ること
    CSV【Comma Separated Values】カンマで区切ったデータ形式 カンマで区切られたテキストデータ
    TSV【Tab-Separeted Values】DBの表や表計算データなど、DB間で情報を交換する方法 Tabで区切られたテキストデータ

プルリクエスト（マージリクエスト）コード変更をレビュワーに通知して、マージを依頼する機能

■ FW／ライブラリ
FW プログラミングにおいて、アプリケーションに必要な一般的な機能があらかじめ実装されたもの。開発効率が上がる
    Spring Framework Javaのオープンソースアプリケーション
    Struts(ストラッツ)JavaWebアプリケーションフレームワーク
    JSF（Java Server Faces）JavaベースのWebアプリケーションフレームワーク

ライブラリ 汎用性の高い複数のプログラムを呼び出して利用する
    Bootstrap フロントエンドWebアプリケーションフレームワーク
    Flyway データベースのマイグレーションツール
    Jackson Javaで使えるJSONライブラリ。JSON形式からJava文字列にマッピング
    Junit Javaプログラムのユニットテストを自動化するためのフレームワーク
    Log4j Javaプログラム用のログAPI。デバッグにも役立つ
    Lombok Java言語におけるボイラープレートコードをソースコードから排除するためのライブラリ。(ボイラープレートコード:言語仕様上省くことができない定型的なコード)
    ORM【Object relational mapping】データベースとオブジェクト指向プログラミング言語間の非互換なデータを変換するプログラミング技法。
